# 분석 기준

## 분석 스코프

- 공항 전체 선택:
  - 해당 공항의 활성 주차장을 합산해서 계산한다.
- 세부 주차장 선택:
  - 선택한 주차장 1개 기준으로 계산한다.

이 스코프는 다음 패널에 동일하게 적용된다.

- 현재 잔여 주차면
- 최근 7일 30분 시계열
- 요일 x 시간 평균 잔여 주차면
- 요일별 시간대 상세 패턴
- 요일별 임계 달성 시간
- 날짜별 임계 달성 시간 히스토리
- 임계치 이벤트

## 시계열

- 기본 범위: 최근 7일
- 기본 간격: 30분
- 계산 방식:
  - 각 30분 버킷 시점까지의 최신 상태를 주차장별로 잡는다.
  - 공항 전체 스코프면 같은 버킷의 주차장별 값을 합산한다.

차트 표시 규칙:

- hover / touch 시 해당 시점의 시간과 잔여 주차면을 툴팁으로 노출한다.
- X축은 6시간 단위(`00:00`, `06:00`, `12:00`, `18:00`)로 라벨링한다.
- 보간이나 smoothing 없이 계단형(step) 라인으로 표시한다.
- 차트는 열리자마자 최신 포인트를 활성화하고, 오른쪽 끝 최신 구간이 먼저 보이도록 스크롤한다.
- 차트 제목 바로 아래에 현재 보고 있는 공항 전체 / 세부 주차장 기준을 명시한다.
- 마지막 포인트 값은 화면 상단 `지금 주차 여유`와 같은 계산 기준을 사용한다.

## 시간대 평균

### 기존 요약

- `GET /parking/analytics/by-hour`
- 최근 N일 관측값을 시간대별로 단순 집계한다.

### 상세 패턴

- `GET /parking/analytics/by-weekday-hour`
- 최근 N일 관측값을 `요일 x 시간`으로 나눠 집계한다.
- 각 셀은 다음 값을 가진다.
  - `average_available_spaces`
  - `min_available_spaces`
  - `max_available_spaces`
  - `observations`
- UI에서는 잔여면이 적을수록 빨간색, 많을수록 파란색으로 그라데이션을 준다.
- 히트맵 위에는 `평균으로 가장 빠듯한 시간`, `평균으로 가장 여유 있는 시간`을 별도 요약으로 보여준다.

## 요일별 패턴

- 화면에서는 `by-weekday-hour` 데이터를 다시 요일 카드로 풀어 보여준다.
- 각 요일 카드에는 다음 정보를 표시한다.
  - 해당 요일 평균 잔여 주차면
  - 가장 빠듯한 시간대
  - 가장 여유가 큰 시간대
  - 24시간 상세 값
- 각 시간 칩도 같은 빨강-파랑 스케일을 사용한다.

## 임계치 이벤트

기본 임계치는 다음 두 개다.

- 10대
- 50대

이벤트 방향:

- `down`: 임계치 미만으로 떨어짐
- `up`: 임계치 이상으로 회복됨

예:

- `50대 미만 진입`
- `10대 이상 회복`

## 임계 달성 시간 인사이트

- `GET /parking/analytics/threshold-insights`
- 최근 관측 시계열을 기준으로 `50대 미만`, `10대 미만` 진입 시점을 다시 계산한다.

화면에서는 두 형태로 보여준다.

- `요일별 임계 달성 시간`
  - 각 요일에서 임계치 아래로 내려가는 대표 시각
  - 현재는 중앙값 기반 대표 시각을 사용한다.
- `날짜별 임계 달성 시간`
  - 실제 날짜별 진입 시각 히스토리
  - 스크롤 가능한 표로 보여준다.

## 수집 데이터와 분석 데이터

- 원본 수집은 `parking_snapshots`에 저장한다.
- 분석은 가능하면 `parking_snapshots`만 보고 계산한다.
- 원본 API 응답 변화에 덜 흔들리도록, 프론트 계산보다 백엔드 집계를 우선한다.
